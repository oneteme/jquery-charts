Documentation jquery-table

Retour à la page de test

jquery-table est un composant Angular Material qui encapsule mat-table avec des fonctionnalités métier prêtes à l'emploi : barre de recherche, tri, groupement, filtres latéraux (slices), colonnes optionnelles, drag & drop de colonnes, pagination, état de chargement et style de ligne personnalisé.

1) Installation & import

import { TableComponent, TableProvider, col } from '@oneteme/jquery-table';

Import dans un composant standalone :

@Component({
  standalone: true,
  imports: [TableComponent],
  template: `<jquery-table [config]="tableConfig" (rowSelected)="onRow($event)"></jquery-table>`,
})

3) API â€” TableCategorySliceProvider<T>

Chaque entrée dans slices définit un panneau de filtres latéral.

Champ Type Description
title string Titre du bloc de filtre affichÃ© dans le panneau.
columnKey string Clé d'une colonne existante : les catégories sont générées automatiquement à partir des valeurs distinctes de la donnée.
categories TableCategoryProvider<T>[] Liste manuelle de catégories (utiliser filter pour des règles personnalisées). Mutuellement exclusif avec columnKey.
showAll boolean Affiche le bouton "Tous" en tête du panneau. Défaut : true.
multiSelect boolean Permet de sélectionner plusieurs catégories simultanément. Défaut : false.

Mode automatique via columnKey

slices: [
  { title: 'Status', columnKey: 'status', showAll: false },
  { title: 'Équipe',  columnKey: 'team',   showAll: true  },
]

Les catégories sont déduites en temps réel des valeurs présentes dans data.

Mode manuel via categories

slices: [
  {
    title: 'Priorité',
    categories: [
      { key: 'high',   label: 'Haute',   filter: (row) => row.priority === 'High' },
      { key: 'medium', label: 'Moyenne', filter: (row) => row.priority === 'Medium' },
      { key: 'low',    label: 'Basse',   filter: (row) => row.priority === 'Low' },
    ],
  },
]

4) Le bouton View

Activé via enableViewButton: true, le bouton View ouvre un menu avec trois sous-menus :

Sous-menu Description Prérequis
Champs Affiche/masque les colonnes. Les colonnes de base sont cochées par défaut (décochables si allowColumnRemoval: true). Les colonnes optionnelles sont décochées par défaut. optional: true sur la colonne ou allowColumnRemoval
Group by Regroupe les lignes selon les valeurs d'une colonne choisie. Un clic sur un groupe le replie/déplie. Re-cliquer sur la colonne active désactive le groupement. columns définies
Slice by Ajoute dynamiquement un filtre latéral basé sur une colonne disponible. Les slices ajoutées dynamiquement apparaissent dans le panneau latéral avec un bouton ×. enableViewButton: true

5) Fonction utilitaire col()

import { col } from '@oneteme/jquery-table';

col<T>(key: string, header: string, sortable?: boolean): TableColumnProvider<T>

Crée rapidement un TableColumnProvider avec sortable: true par défaut et la résolution automatique de valeur via field(key) de jquery-core. 

columns: [
  col<RepoRow>('issue',  'Issue'),
  col<RepoRow>('status', 'Status'),
  col<RepoRow>('owner',  'Owner', false), // non triable
]

6) Style de ligne personnalisé — rowClass

La fonction rowClass reçoit la ligne brute T et son index, et retourne une valeur compatible ngClass appliquée sur le <tr>. 

rowClass: (row: RepoRow, index: number) => {
  if (row.status === 'Done')        return 'row-done';
  if (row.status === 'In Progress') return 'row-in-progress';
  return '';
}

Exemple CSS correspondant (à placer dans le composant consommateur) :

::ng-deep tr.mat-mdc-row {
  &.row-done > td.mat-mdc-cell:first-of-type {
    border-left: 3px solid #4caf50;
    background: linear-gradient(90deg, rgba(76,175,80,0.08) 0%, transparent 100%);
  }
  &.row-in-progress > td.mat-mdc-cell:first-of-type {
    border-left: 3px solid #0080ff;
    background: linear-gradient(90deg, rgba(0,128,255,0.08) 0%, transparent 100%);
  }
}

Les trois formes de retour acceptées :

  • string — une classe CSS : 'row-error'
  • string[] — plusieurs classes : ['row-error', 'row-bold']
  • Record<string, boolean> — conditionnel : { 'row-error': true, 'row-bold': false }

8) Exemple complet

interface RepoRow {
  issue: string;
  status: 'Backlog' | 'In Progress' | 'Done';
  owner: string;
  priority: 'Low' | 'Medium' | 'High';
  team: string;
  updatedAt: string;
}

tableConfig: TableProvider<RepoRow> = {
  title: 'Ticket board',
  enableSearchBar: true,
  enableViewButton: true,
  enableColumnDragDrop: true,
  allowColumnRemoval: true,
  enablePagination: true,
  pageSize: 5,
  pageSizeOptions: [5, 10, 20],
  columns: [
    col<RepoRow>('issue',  'Issue'),
    col<RepoRow>('status', 'Status'),
    col<RepoRow>('owner',  'Owner'),
    { ...col<RepoRow>('priority',  'Priorité'),             optional: true },
    { ...col<RepoRow>('team',      'Équipe'),               optional: true },
    { ...col<RepoRow>('updatedAt', 'Dernière mise à jour'), optional: true },
  ],
  slices: [
    { title: 'Status', columnKey: 'status', showAll: false },
  ],
  rowClass: (row) => {
    if (row.status === 'Done')        return 'row-done';
    if (row.status === 'In Progress') return 'row-in-progress';
    return '';
  },
  emptyStateLabel: 'Aucun résultat',
  loadingStateLabel: 'Chargement...',
  data: rows,
};

<jquery-table
  [config]="tableConfig"
  [isLoading]="isLoading"
  (rowSelected)="onRowSelected($event)"
></jquery-table>
