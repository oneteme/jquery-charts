Cette page explique comment utiliser jquery-table en restant compatible avec Angular Material, tout en ajoutant les fonctionnalités métier (slice catégorie, colonnes optionnelles, etc.).

1) Angular Material classique vs wrapper jquery-table

Implémentation Material classique

  • Vous gérez explicitement MatTableDataSource, pagination, tri et colonnes.
  • Vous implémentez vous-même le filtre latéral et les compteurs de catégorie.
  • Vous ajoutez manuellement les actions d’ajout/retrait de colonnes.

Implémentation avec jquery-table

  • Pagination interne en MatTableDataSource + MatPaginator natifs.
  • Configuration unifiée via TableProvider<T>.
  • Features métier prêtes à l’emploi (slice, colonnes optionnelles, events).

2) Modèle de données métier: RepoRow

RepoRow représente une ligne fonctionnelle de votre tableau. Ce modèle est votre source de vérité métier. 

interface RepoRow {
  issue: string;
  status: 'Backlog' | 'In Progress' | 'Done';
  owner: string;
  priority: 'Low' | 'Medium' | 'High';
  team: string;
  updatedAt: string;
}

3) API TableProvider<T>

Tous les champs de TableProvider sont techniquement optionnels côté TypeScript, mais certains sont recommandés pour un comportement complet.

Champ Optionnel Natif Material Usage
columns Oui (recommandé) Non Colonnes de base affichées par défaut.
data Oui (recommandé) Oui (concept dataSource) Données à afficher dans la table.
title Oui Non Titre de l’exemple/tableau.
optionalColumns / availableColumns Oui Non Colonnes affichées dans la section « Colonnes optionnelles » du menu Colonnes.
showAddColumnButton Oui Non Affiche le bouton + ouvrant le menu « Colonnes » (checkbox).
categorySlice Oui Non Panneau gauche (catégories + filtres).
enablePagination / pageSize / pageSizeOptions Oui Oui Paramètres de pagination Material.
isLoading / loadingStateLabel Oui Non (ajout wrapper) État de chargement et texte affiché.
emptyStateLabel Oui Non (ajout wrapper) Message si aucune donnée.

4) Champs natifs vs champs wrapper

Entrée composant Type Nature Remarque
dataSource T[] | { data: T[] } Natif-compatible Compatible migration Material.
displayedColumns string[] Natif-compatible Même logique que mat-table.
columnLabels Record<string, string> Wrapper utilitaire Libellés rapides sans mapping manuel.
config TableProvider<T> Wrapper principal Source de configuration avancée.
isLoading boolean Wrapper Affichage “chargement” piloté.

5) Sur quelle base construire categorySlice ?

categorySlice se construit à partir d’un champ métier de RepoRow qui segmente naturellement vos données (ex: status, priority, team). Dans l’exemple, la base choisie est status.

  • 1 catégorie = 1 règle de filtre filter(row) => boolean.
  • Les compteurs affichés proviennent du dataset courant (data).
  • Le libellé utilisateur est indépendant de la valeur technique.
  • Si votre métier change, vous adaptez seulement les règles de categories.
categorySlice: {
  title: 'Status',
  allLabel: 'Tous',
  categories: [
    { key: 'backlog', label: 'Backlog', filter: (row) => row.status === 'Backlog' },
    { key: 'in-progress', label: 'In Progress', filter: (row) => row.status === 'In Progress' },
    { key: 'done', label: 'Done', filter: (row) => row.status === 'Done' },
  ],
}

6) Exemple complet recommandé

const tableConfig: TableProvider<RepoRow> = {
  title: 'Ticket board',
  showAddColumnButton: true,
  enablePagination: true,
  pageSize: 5,
  columns: [
    col<RepoRow>('issue', 'Issue'),
    col<RepoRow>('status', 'Status'),
    col<RepoRow>('owner', 'Owner'),
  ],
  optionalColumns: [
    col<RepoRow>('priority', 'Priorité'),
    col<RepoRow>('team', 'Équipe'),
  ],
  categorySlice: {
    title: 'Status',
    allLabel: 'Tous',
    categories: [...],
  },
  data: rows,
};
<jquery-table
  [config]="tableConfig"
  [isLoading]="isLoading"
  (rowSelected)="onRowSelected($event)"
></jquery-table>

Le bouton + ouvre le menu Colonnes avec deux groupes: Colonnes puis Colonnes optionnelles. Chaque entrée est une checkbox pour afficher/masquer la colonne correspondante.

7) Résumé opérationnel

  • TableProvider centralise la config métier.
  • RepoRow formalise clairement les données passées à la table.
  • Vous gardez les bases Angular Material tout en ajoutant des capacités spécifiques au wrapper.
  • La migration est progressive: mode natif-compatible ou mode config avancée.
