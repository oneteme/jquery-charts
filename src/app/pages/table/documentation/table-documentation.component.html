Documentation jquery-table

Composant Angular Material encapsulant mat-table avec recherche, tri, groupement, slices, colonnes optionnelles, drag & drop et pagination. Le composant <slice-panel> est également disponible de façon indépendante pour filtrer graphiques, listes…

1) Installation & import

import { TableComponent, SlicePanelComponent, TableProvider, SliceConfig, SliceCategory, col } from '@oneteme/jquery-table';

Import dans un composant standalone :

@Component({
  standalone: true,
  imports: [TableComponent],           // pour le tableau
  // imports: [SlicePanelComponent],   // pour le panneau seul (graphique, etc.)
  template: `<jquery-table [config]="cfg" (rowSelected)="onRow($event)"></jquery-table>`,
})

2) API — TableProvider<T>

Toute la configuration du tableau se passe via l'entrée [config]. Les données s'injectent via [data] (séparé de la config pour faciliter le rechargement), et l'état de chargement via [isLoading].

Inputs du composant

InputTypeDéfautDescription
[config] TableProvider<T> — Configuration principale : colonnes, slices, search, pagination, view, labels…
[data] T[] [] Données à afficher. Peut changer indépendamment de config.
[isLoading] boolean false Affiche un overlay « Chargement » par-dessus le tableau.

Champs de TableProvider<T>

ChampTypeDescription
columns TableColumnProvider<T>[] Colonnes affichées. Voir section 6 pour TableColumnProvider et col().
title string Titre affiché dans la barre supérieure du tableau.
slices SliceConfig<T>[] Filtres catégoriels latéraux (panneau slice). Voir section 3.
enableSliceToggle boolean Défaut : true. Bouton pour replier/déplier le panneau slice.
search TableSearchConfig Configure la barre de recherche. Voir sous-section ci-dessous.
pagination TablePaginationConfig Configure la pagination. Voir sous-section ci-dessous.
view TableViewConfig Configure le bouton View (Champs / Group by / Slice by). Voir section 5.
labels TableLabelsConfig Messages d'état : vide et chargement. Voir sous-section ci-dessous.
defaultSort { active: string; direction: 'asc' | 'desc' } Tri initial appliqué au chargement. Non écrasé lors des changements de données.
rowClass (row: T, index: number) => string | string[] | Record<string, boolean> Classe(s) CSS appliquée(s) dynamiquement sur chaque <tr>. Voir section 7.
export TableExportConfig<T> Configure le bouton Export CSV dans la toolbar. Voir sous-section ci-dessous.

TableSearchConfig

ChampTypeDéfautDescription
enabled boolean false Active la barre de recherche texte globale.
initialQuery string — Terme de recherche pré-rempli à l'initialisation.
searchColumns string[] — Restreint la recherche aux colonnes dont la key est dans ce tableau. Si absent, toutes les colonnes sont cherchées. 
search: { enabled: true, searchColumns: ['name', 'status'] }

TablePaginationConfig

ChampTypeDéfautDescription
enabled boolean true Active la pagination. Désactivez pour afficher toutes les lignes (déconseillé sur grands jeux de données).
pageSize number 5 Nombre de lignes par page.
pageSizeOptions number[] [5, 10, 20] Options du sélecteur de taille de page.
pageSizeOptionsGroupBy number[] — Options spécifiques au mode Group by. Si absent, utilise pageSizeOptions. 
pagination: { enabled: true, pageSize: 10, pageSizeOptions: [5, 10, 25] }

TableViewConfig

ChampTypeDéfautDescription
enabled boolean false Affiche le bouton View. Nécessaire pour Champs, Group by et Slice by.
enableColumnRemoval boolean true Autorise à masquer les colonnes de base via View > Champs.
enableColumnDragDrop boolean false Poignée sur chaque en-tête pour réordonner les colonnes par glisser-déposer. 
view: { enabled: true, enableColumnRemoval: true, enableColumnDragDrop: true }

TableLabelsConfig

ChampTypeDéfautDescription
empty string 'Aucune donnée' Message affiché quand le tableau est vide (ou qu'aucun filtre ne retourne de résultat).
loading string 'Chargement des données...' Message affiché pendant [isLoading]="true". 
labels: { empty: 'Aucun résultat', loading: 'Chargement...' }

TableExportConfig<T>

ChampTypeDéfautDescription
enabled boolean false Affiche le bouton Exporter dans la toolbar.
filename string 'export' Nom du fichier généré (sans extension). L'extension .csv est ajoutée automatiquement.
transform (row: T) => Record<string, string> — Transforme chaque ligne avant export. Si absent, les valeurs affichées sont utilisées.

L'export ne porte que sur les données filtrées (slice + recherche) et les colonnes visibles. Le fichier est encodé en UTF-8 avec BOM pour une compatibilité Excel correcte. 

export: { enabled: true, filename: 'tickets-2024' }

// Avec transformation personnalisée
export: {
  enabled: true,
  filename: 'rapport',
  transform: (row) => ({ issue: row.issue, statut: row.status === 'Done' ? 'Terminé' : 'En cours' })
}

3) API — SliceConfig<T>

Type exporté par @oneteme/jquery-table. Chaque entrée de slices (ou de [sliceConfigs] sur <slice-panel>) définit un bloc de filtres.

ChampTypeDescription
title string Libellé du bloc affiché dans le panneau.
columnKey string Catégories générées automatiquement depuis les valeurs distinctes de la clé.
Mutuellement exclusif avec categories.
categories SliceCategory<T>[] Catégories manuelles avec filter personnalisé. Voir section 4.
Mutuellement exclusif avec columnKey.
multiSelect boolean Sélection multiple simultanée. Défaut : false.

Mode automatique — columnKey

slices: [
  { title: 'Status', columnKey: 'status' },
  { title: 'Equipe',  columnKey: 'team', multiSelect: true },
]

Les catégories sont déduites en temps réel des valeurs présentes dans data.

Mode manuel — categories

slices: [
  {
    title: 'Priorite',
    categories: [
      { key: 'high',   label: 'Haute',   filter: (row) => row.priority === 'High'   },
      { key: 'medium', label: 'Moyenne', filter: (row) => row.priority === 'Medium' },
      { key: 'low',    label: 'Basse',   filter: (row) => row.priority === 'Low'    },
    ],
  },
]

4) API — SliceCategory<T>

Une catégorie cliquable dans un bloc de filtre.

ChampTypeDescription
key string Identifiant unique (détecte les sélections actives).
label string Texte affiché dans le bouton.
filter (row: T) => boolean Prédicat déterminant si une ligne appartient à cette catégorie.

5) Le bouton View

Activé via view: { enabled: true }.

Sous-menuDescriptionPrérequis
Champs Affiche/masque les colonnes.
• Colonnes de base : cochées par défaut, décochables si view.enableColumnRemoval: true.
• Colonnes optional: true : décochées par défaut, toujours activables. 		view: { enabled: true }
Group by Regroupe les lignes par valeur de colonne. Clic sur un groupe : replie/déplie. Re-clic sur la même colonne : désactive. Compatible avec pagination: { enabled: false }. view: { enabled: true }
Slice by Ajoute dynamiquement un filtre latéral basé sur une colonne. Apparait dans le panneau avec un bouton × pour le retirer. view: { enabled: true }

Exemples

// Bouton View minimal (Champs + Group by + Slice by)
view: { enabled: true }

// Avec suppression de colonnes de base autorisée
view: { enabled: true, enableColumnRemoval: true }

// Avec glisser-déposer des colonnes
view: { enabled: true, enableColumnRemoval: true, enableColumnDragDrop: true }

6) Fonction utilitaire col()

import { col } from '@oneteme/jquery-table';

col<T>(key: string, header: string, sortable = true): TableColumnProvider<T>

Crée un TableColumnProvider avec sortable: true par défaut et résolution automatique de la valeur via field(key). 

columns: [
  col<Row>('name',   'Nom'),
  col<Row>('status', 'Statut'),
  col<Row>('date',   'Date', false),               // non triable
  { ...col<Row>('team', 'Equipe'), optional: true }, // masquee par defaut (enableViewButton requis)
]

7) Style de ligne — rowClass

Fonction appelée pour chaque ligne. Retourne une valeur compatible [ngClass] appliquée sur le <tr>.

Les trois formes acceptées :

  • string — une classe : 'row-error'
  • string[] — plusieurs classes : ['row-error', 'row-bold']
  • Record<string, boolean> — conditionnel : { 'row-error': true }
rowClass: (row: Row) => {
  if (row.status === 'Done')        return 'row-done';
  if (row.status === 'In Progress') return 'row-in-progress';
  return '';
}
// styles a placer dans le composant consommateur
::ng-deep tr.mat-mdc-row {
  &.row-done > td.mat-mdc-cell:first-of-type {
    border-left: 3px solid #4caf50;
    background: linear-gradient(90deg, rgba(76,175,80,.08) 0%, transparent 100%);
  }
  &.row-in-progress > td.mat-mdc-cell:first-of-type {
    border-left: 3px solid #0080ff;
    background: linear-gradient(90deg, rgba(0,128,255,.08) 0%, transparent 100%);
  }
}

8) Outputs — <jquery-table>

EvénementTypeDescription
(rowSelected) EventEmitter<T> Emis au clic sur une ligne (pas sur les en-têtes de groupe).
(sortChange) EventEmitter<{ active: string; direction: 'asc'|'desc'|'' }> Emis à chaque changement de tri (colonne + direction).
(pageChange) EventEmitter<{ pageIndex: number; pageSize: number }> Emis à chaque changement de page ou de taille de page.
(searchChange) EventEmitter<string> Emis à chaque frappe dans la barre de recherche.
(groupByChange) EventEmitter<string | null> Emis quand le Group by est activé (clé colonne) ou désactivé (null).
(columnsChange) EventEmitter<string[]> Emis quand la liste des colonnes visibles change (ajout, suppression, réordonnancement). Payload : clés dans leur ordre d'affichage.
(columnAdded) EventEmitter<TableColumnProvider<T>> Colonne ajoutée aux colonnes actives.
(columnRemoved) EventEmitter<TableColumnProvider<T>> Colonne retirée des colonnes actives.

Les outputs d'état (sortChange, pageChange, searchChange, groupByChange, columnsChange) sont utiles pour persister l'état dans l'URL ou restaurer une session.

9) Composant standalone — <slice-panel>

SlicePanelComponent peut s'utiliser indépendamment du tableau pour filtrer n'importe quelle donnée (graphique, liste…). Ne pas passer [columns] en dehors du contexte tableau.

Inputs

InputTypeDéfautDescription
[sliceConfigs] SliceConfig<T>[] [] Définition des blocs de filtres. Voir section 3.
[data] T[] [] Source brute : calcul des compteurs et des catégories automatiques (columnKey).
[columns] TableColumnProvider<T>[] [] Colonnes pour le menu Slice by dynamique.
Optionnel — ne pas passer dans un contexte graphique.
[showToggle] boolean true Bouton pour replier/déplier le panneau.
[showCounts] boolean true Nombre d'éléments par catégorie.
[collapsedByDefault] boolean false Tous les blocs repliés à l'initialisation.
[alwaysShow] boolean false Panneau visible même sans filtre configuré.

Outputs

EvénementTypeDescription
(filterChange) EventEmitter<(row: T) => boolean> Prédicat combiné prêt à l'emploi : data.filter(predicate).
(activeKeysChange) EventEmitter<string[][]> Clés actives par bloc. [i] = clés sélectionnées pour le bloc i. Utile pour un récapitulatif des filtres.
(dynamicSliceKeysChange) EventEmitter<string[]> Clés des colonnes ajoutées via Slice by.

9b) Prérequis de rendu — hauteur parent

<jquery-table> distribue la hauteur reçue via flex-column interne. Si aucune hauteur n'est fournie par le parent, le corps du tableau s'effondre à 0px et les lignes deviennent invisibles (seuls le header de toolbar et le paginator restent visibles).

Deux approches selon le contexte :

A) Hauteur fixe (page de démo, carte, modale)

/* SCSS du composant parent */
jquery-table {
  display: block;   /* l'élément est inline par défaut */
  height: 460px;    /* hauteur suffisante pour afficher les lignes */
}

B) Hauteur fluide — remplissage de la fenêtre (tableau plein écran)

/* SCSS du composant parent — pattern recommandé */
:host {
  display: flex;
  flex-direction: column;
  height: 100%;      /* le parent (ex. mat-drawer-content) doit lui-même avoir une hauteur */
  overflow: hidden;
}

jquery-table {
  flex: 1 1 0;       /* occupe tout l'espace restant après les autres éléments */
  min-height: 0;     /* autorise le flex-item à se réduire en-deçà de son contenu */
}

Ce second pattern suppose que la chaîne de conteneurs ancêtres fournit une hauteur concrète jusqu'à la racine (ex. html, body { height: 100%; } ou un layout Angular Material avec mat-sidenav-content qui impose sa propre hauteur).

10) Exemple complet — Tableau

import { col, TableProvider, JQT_I18N } from '@oneteme/jquery-table';

interface Row {
  issue: string;
  status: 'Backlog' | 'In Progress' | 'Done';
  owner: string;
  priority: 'Low' | 'Medium' | 'High';
  team: string;
}

// i18n optionnel (défauts en français)
providers: [{ provide: JQT_I18N, useValue: { emptyState: 'Aucun ticket trouvé' } }]

readonly cfg: TableProvider<Row> = {
  title: 'Ticket board',

  // Recherche texte restreinte à deux colonnes
  search: { enabled: true, searchColumns: ['issue', 'owner'] },

  // Le bouton View active Champs, Group by et Slice by dynamique
  view: {
    enabled: true,
    enableColumnRemoval: true,   // autorise à masquer les colonnes de base
    enableColumnDragDrop: true,  // réordonnement par glisser-déposer
  },

  // Pagination
  pagination: { enabled: true, pageSize: 10, pageSizeOptions: [5, 10, 25] },

  // Messages d'état (prend le dessus sur JQT_I18N)
  labels: { empty: 'Aucun ticket', loading: 'Chargement des tickets...' },

  // Export CSV des données filtrées
  export: { enabled: true, filename: 'tickets' },

  // Tri initial
  defaultSort: { active: 'priority', direction: 'desc' },

  columns: [
    col<Row>('issue',  'Issue'),
    col<Row>('status', 'Statut'),
    col<Row>('owner',  'Responsable'),
    // optional: true => masquée par défaut, activable via View > Champs
    col<Row>('priority', 'Priorité', { optional: true }),
    col<Row>('team',     'Équipe',   { optional: true }),
  ],

  slices: [
    { title: 'Statut', columnKey: 'status' },
    {
      title: 'Priorité',
      categories: [
        { key: 'high',   label: 'Haute',   filter: r => r.priority === 'High'   },
        { key: 'medium', label: 'Moyenne', filter: r => r.priority === 'Medium' },
        { key: 'low',    label: 'Basse',   filter: r => r.priority === 'Low'    },
      ],
    },
  ],

  rowClass: r => {
    if (r.status === 'Done')        return 'row-done';
    if (r.status === 'In Progress') return 'row-in-progress';
    return '';
  },
};
<!-- data et isLoading sont des Inputs séparés de config -->
<jquery-table
  [config]="cfg"
  [data]="rows"
  [isLoading]="loading"
  (rowSelected)="onRow($event)">
</jquery-table>

11) Personnalisation visuelle — CSS custom properties

Toutes les couleurs principales du composant sont exposées via des CSS custom properties sur :host. Aucun ::ng-deep, aucun !important n'est nécessaire côté consommateur — il suffit de surcharger les variables depuis votre feuille de styles globale. 

/* styles.scss (ou tout fichier global) */
jquery-table {
  /* ── Texte ───────────────────────────────────────────── */
  --jqt-title-color:        #1f2937;  /* titre du tableau (h3)                    */
  --jqt-text-color:         #111827;  /* texte principal (cellules, labels)        */
  --jqt-text-secondary:     #6b7280;  /* texte secondaire (badges, infos)          */
  --jqt-text-muted:         #9ca3af;  /* placeholder, icônes discrètes              */

  /* ── Surfaces générales ─────────────────────────────── */
  --jqt-surface-color:      #ffffff;  /* fond barre recherche, boutons, panel      */
  --jqt-surface-alt-color:  #f9fafb;  /* fond alternatif subtil                    */
  --jqt-row-hover-bg:       #f3f4f6;  /* survol d'une ligne / fond badge           */

  /* ── Cellules (Angular Material) ─────────────────────── */
  --jqt-header-bg:          transparent; /* fond de la ligne d'en-tête            */
  --jqt-header-text-color:  #374151;     /* texte des cellules d'en-tête           */
  --jqt-cell-bg:            transparent; /* fond des cellules de corps               */

  /* ── Footer / paginator ──────────────────────────────── */
  --jqt-footer-bg:          #ffffff;  /* fond de la barre de pagination            */

  /* ── Bordures ────────────────────────────────────────── */
  --jqt-border-color:       #e5e7eb;  /* bordures du tableau et éléments UI     */

  /* ── Lignes de groupe (groupBy) ──────────────────────── */
  --jqt-group-bg:           #eef2f7;  /* fond des lignes de groupe                 */
  --jqt-group-bg-hover:     #e4ecf4;  /* fond au survol d'une ligne de groupe      */
