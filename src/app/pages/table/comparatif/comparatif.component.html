mat-table vs. jquery-table

Même dataset, même besoin : recherche, pagination, tri, filtres catégoriels et colonnes réorganisables. Voici ce que chaque solution demande d’écrire.

Angular Material (mat-table seul)

  • Affichage des données
  • Tri via MatSortModule
  • Recherche → champ + filtre à coder manuellement
  • Pagination → MatPaginatorModule + DataSource custom
  • Filtres catégoriels → logique entièrement à écrire
  • Colonnes masquables → ngIf sur chaque colonne
  • Drag & drop colonnes → DragDropModule + gestion d’état

Code nécessaire (extrait)

{{ matCode }}

jquery-table

  • Affichage des données
  • Tri intégré (sortable: true)
  • Recherche intégrée (enableSearchBar: true)
  • Pagination intégrée (enablePagination: true)
  • Filtres catégoriels (slices: [...])
  • Colonnes masquables (allowColumnRemoval: true)
  • Drag & drop colonnes (enableColumnDragDrop: true)

Code nécessaire (complet)

{{ jqCode }}

Démo live

Recherche, pagination, tri, slices latéraux, masquage et réorganisation des colonnes — tout configuré avec l’objet tableConfig ci-dessus.
